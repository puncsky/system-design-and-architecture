1. 同理心 / 视角转换是最重要的。
    1. 意识到代码首先是为人类阅读而编写的，然后才是为机器执行。
    2. 软件是如此“柔软”，有很多方法可以实现同一目标。关键在于做出适当的权衡，以满足需求。
    3. 发明与简化：Apple Pay RFID 与 微信扫码二维码。

2. 选择可持续的架构，以减少每个功能的人力资源成本。

3. 采用模式和最佳实践。

4. 避免反模式
    * 缺少错误处理
    * 回调地狱 = 意大利面条代码 + 不可预测的错误处理
    * 过长的继承链
    * 循环依赖
    * 过于复杂的代码
        * 嵌套的三元操作
        * 注释掉未使用的代码
    * 缺少国际化，特别是 RTL 问题
    * 不要重复自己
      * 简单的复制粘贴
      * 不合理的注释

5. 有效的重构
    * 语义版本
    * 永远不要对非主要版本引入破坏性更改
        * 两腿变更
